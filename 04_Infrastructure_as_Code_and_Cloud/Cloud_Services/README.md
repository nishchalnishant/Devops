# Cloud Interview Track

This section is the cloud-specific path inside the repository. The current depth is strongest for Azure, with an additional Azure-to-AWS mapping file so you can translate concepts across providers during interviews.

In the new learning structure, this section is used mainly in Phase 2 for cloud and platform fundamentals, and again in Phase 4 for senior-level cloud design and operations trade-offs.

## What This Section Covers

- Azure fundamentals, governance, IAM, networking, storage, compute, AKS, monitoring, and security
- Cross-cloud translation between Azure and AWS service names and patterns
- Design, troubleshooting, high availability, and cost-awareness questions

## Recommended Study Order

1. `azure-easy-qeustions.md`
2. `azure-medium-questions.md`
3. `azure-hard-questions.md`
4. `azure-aws-similarities.md`
5. `../cheat sheet/azure cloud.pdf` for final revision

## High-Value Topics To Revise

- Subscription, resource group, and policy structure
- Azure RBAC and identity management
- Virtual networks, subnets, NSGs, private endpoints, and DNS
- VMs, scale sets, App Service, ACR, ACI, and AKS
- Storage types and when to use each one
- Azure Monitor, Defender, Sentinel, and service health
- Disaster recovery, multi-region design, and traffic routing

## Interview Advice

- Learn the Azure-native name and the cross-cloud equivalent.
- Explain trade-offs using reliability, security, cost, and operability.
- Be ready to compare AKS with EKS or self-managed Kubernetes.
- For design questions, mention availability zones, identity boundaries, private networking, monitoring, and rollback.
